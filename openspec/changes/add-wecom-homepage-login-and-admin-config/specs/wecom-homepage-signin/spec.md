## ADDED Requirements

### Requirement: 登录首页必须展示企业微信扫码登录入口
当应用启用了可见的 WeCom 登录 Provider 时，登录首页 MUST 向用户展示明确的企业微信登录入口，而不是仅保留通用 OAuth 跳转按钮。

#### Scenario: 应用启用了企业微信登录
- **WHEN** 登录页加载完成且当前应用存在启用中的 WeCom Provider
- **THEN** 页面必须展示企业微信登录入口，并允许用户切换到企业微信扫码登录视图

#### Scenario: 应用未启用企业微信登录
- **WHEN** 登录页加载完成且当前应用不存在可用的 WeCom Provider
- **THEN** 页面不得展示误导性的企业微信扫码登录入口

### Requirement: 企业微信登录视图必须支持扫码容器与显式发起回退
系统 MUST 在企业微信登录视图中优先提供扫码容器，并在二维码组件不可用或配置不完整时，提供明确的失败反馈或显式发起登录动作。

#### Scenario: 企业微信扫码组件可用
- **WHEN** 登录页成功获取企业微信扫码配置且前端能够加载官方扫码组件
- **THEN** 页面必须渲染可扫码的企业微信二维码容器，并展示当前登录引导文案

#### Scenario: 企业微信扫码组件不可用
- **WHEN** 前端无法加载企业微信扫码组件脚本或二维码容器初始化失败
- **THEN** 页面必须展示可读错误提示，并保留用户可触发的企业微信登录发起入口

#### Scenario: 企业微信登录未完成配置
- **WHEN** 当前应用存在 WeCom Provider 但关键登录参数缺失
- **THEN** 页面必须展示“未完成企业微信登录配置”类提示，而不是静默空白或跳转到不可用地址

### Requirement: 企业微信扫码登录回调结果必须转化为首页可理解的反馈
系统 MUST 将企业微信扫码登录成功、失败、取消或配置异常的结果转化为首页或回调页可理解的状态反馈，并为失败场景保留重试路径。

#### Scenario: 企业微信扫码登录成功
- **WHEN** 企业微信完成授权并返回有效回调参数，且后端成功换取并绑定当前用户身份
- **THEN** 系统必须完成登录态建立，并跳转到既定的登录成功目标页面

#### Scenario: 企业微信扫码登录失败
- **WHEN** 企业微信回调缺少有效授权参数，或后端无法完成用户信息换取
- **THEN** 系统必须向用户展示明确失败原因或通用错误提示，并允许用户重新发起登录

#### Scenario: 企业微信账号需要额外处理
- **WHEN** 企业微信回调用户存在未绑定、未开通或无法直接建立登录态的情况
- **THEN** 系统必须返回可区分的反馈状态，而不是将其误判为普通登录成功
