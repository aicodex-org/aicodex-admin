## ADDED Requirements

### Requirement: 后台必须提供面向企业微信网页登录的明确配置项
后台 Provider 配置页 MUST 为 WeCom 登录场景提供清晰的字段表达，至少覆盖登录模式、应用标识、密钥、Agent ID、Scope 和回调地址相关信息。

#### Scenario: 管理员选择企业微信 Provider
- **WHEN** 管理员在后台新建或编辑 `WeCom` 类型 Provider
- **THEN** 页面必须展示企业微信登录所需的关键字段，并对字段用途给出明确标签或说明

#### Scenario: 管理员需要查看回调地址
- **WHEN** 管理员配置企业微信登录 Provider
- **THEN** 页面必须能够向管理员展示当前系统要求配置到企业微信后台的回调地址或等效联调信息

### Requirement: 后台必须按企业微信登录模式执行字段校验
系统 MUST 根据 `Internal` / `Third-party` 与 `Normal` / `Silent` 的组合，对企业微信登录配置执行不同的必填约束与提示，避免管理员保存不可用配置。

#### Scenario: Internal Normal 模式保存配置
- **WHEN** 管理员选择 `Internal` 且 `Normal` 的企业微信登录模式
- **THEN** 系统必须校验该模式下所需的关键字段已填写，并在缺失时阻止保存或给出明确错误提示

#### Scenario: Internal Silent 模式保存配置
- **WHEN** 管理员选择 `Internal` 且 `Silent` 的企业微信登录模式
- **THEN** 系统必须按照该模式要求校验字段，并提示其适用场景与限制

#### Scenario: Third-party 模式保存配置
- **WHEN** 管理员选择 `Third-party` 企业微信登录模式
- **THEN** 系统必须按照第三方服务商网页登录要求校验对应字段，并避免沿用内部应用模式的误导性提示

### Requirement: 后台必须向管理员提供企业微信登录联调提示
系统 MUST 为企业微信登录配置提供最小必要的联调与排障信息，帮助管理员完成企业微信后台配置并快速定位常见错误。

#### Scenario: 管理员完成企业微信配置
- **WHEN** 管理员查看或保存企业微信登录 Provider
- **THEN** 页面必须展示或链接到关于可信域名、授权回调、Agent ID、CorpId、Secret 和 Scope 的最小必要说明

#### Scenario: 配置可能导致首页无法发起扫码登录
- **WHEN** 管理员保存了缺少关键参数或模式与字段不匹配的企业微信登录配置
- **THEN** 系统必须给出可操作的提示，说明哪些字段会影响首页企业微信扫码登录

### Requirement: 后端必须按保存的企业微信配置一致地执行登录链路
后端企业微信登录实现 MUST 根据后台保存的 Provider 配置，稳定地构造授权链路、解析回调并换取用户信息，不得出现前后端协议不一致。

#### Scenario: 前端根据后台配置发起企业微信登录
- **WHEN** 登录首页基于当前 Provider 配置发起企业微信登录
- **THEN** 后端必须能够按相同模式和参数解析回调，并完成用户身份换取

#### Scenario: 后台配置与后端接口口径不一致
- **WHEN** 企业微信登录所需的配置组合无法支撑当前后端登录链路
- **THEN** 系统必须在保存、联调或回调阶段返回可识别的错误，而不是让错误静默发生
